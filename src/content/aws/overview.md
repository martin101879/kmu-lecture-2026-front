# 배포 방식

## 배포란?

개발한 애플리케이션을 사용자가 접근할 수 있는 서버에 올리는 것입니다.

```
로컬 개발:     localhost:8080 (내 PC에서만 접근 가능)
    ↓ 배포
서버 배포:     https://api.example.com (누구나 접근 가능)
```

## AWS 배포 옵션

AWS에서 애플리케이션을 배포하는 방법은 여러 가지가 있습니다. 크게 **서버를 직접 관리하는 방식**과 **AWS가 서버를 대신 관리하는 방식** (서버리스)으로 나뉩니다.

| | 서버 기반 | 서버리스 |
|---|---|---|
| 서버 관리 | 직접 관리 (OS, 패치 등) | AWS가 관리 |
| 과금 | 서버가 켜져 있는 동안 과금 | 사용한 만큼만 과금 |
| 스케일링 | 수동 또는 설정 필요 | 자동 |
| 대표 서비스 | EC2, ECS on EC2 | Fargate, Lambda |

### 주요 옵션 비교

| | **EC2** | Elastic Beanstalk | ECS Fargate | EKS | Lambda |
|---|---|---|---|---|---|
| 설명 | **가상 서버를 빌려서 직접 관리** | 코드 업로드하면 인프라 자동 구성 | Docker 컨테이너 실행 (서버리스) | Kubernetes 기반 컨테이너 관리 | 함수 단위 실행 |
| 서버 관리 | **직접 관리** | 반자동 | 서버리스 | 선택 | 완전 서버리스 |
| 배포 방식 | **Docker (이 강의)** | JAR 업로드 | Docker | Docker | 함수 코드 |
| ALB/HTTPS | **직접 설정** | 자동 | CloudFormation으로 구성 | 직접 설정 | API Gateway |
| 스케일링 | **수동** | 자동 | 자동 | 자동 | 자동 |
| Spring Boot | **O** | O | O | O | X |
| 적합한 규모 | **소~중규모** | 중규모 | 소~대규모 | 대규모 | 이벤트 처리 |

> Lambda는 요청이 올 때마다 함수를 실행하고 종료하는 **호출-응답형** 구조입니다. Spring Boot처럼 서버가 상시 실행되는 환경과 맞지 않습니다 (매 호출마다 Spring 초기화 → Cold Start 발생). EKS는 Kubernetes 기반으로 대규모 마이크로서비스에 적합하지만, 학습 곡선이 높아 소규모 프로젝트에는 과합니다.

## 프론트엔드 배포 옵션

React 앱은 `npm run build`로 **정적 파일** (HTML, CSS, JS)을 생성합니다. 이 파일을 사용자에게 서빙하는 방법은 여러 가지가 있습니다.

- **백엔드 서버에 포함** — 빌드 결과물을 Spring Boot의 `static/` 폴더에 넣어 하나의 서버에서 API + 정적 파일을 모두 서빙
- **웹서버 배포** — Nginx 같은 웹서버에서 정적 파일을 서빙. EC2에 직접 설치하거나 Docker로 실행
- **S3 + CloudFront** — 정적 파일을 S3에 업로드하고 CloudFront (CDN)로 전 세계에 배포

| | 백엔드에 포함 | 웹서버 (Nginx) | S3 + CloudFront |
|---|---|---|---|
| 서버 관리 | 백엔드와 동일 | 직접 관리 | **불필요** |
| CDN | 없음 | 별도 구성 | **자동** |
| HTTPS | 백엔드 설정 | 직접 설정 | **자동** |
| 프론트/백엔드 분리 배포 | 불가 | 가능 | **가능** |
| CORS | 불필요 | 필요할 수 있음 | 필요 |
| 적합한 경우 | 소규모, 프로토타입 | 커스텀 서버 설정 필요 시 | **대부분의 SPA** |

## 이 강의에서의 선택

| 영역 | 선택 | 이유 |
|------|------|------|
| 백엔드 | **EC2 + Docker + ALB** | 서버에 직접 Docker 실행, 배포 과정을 직접 체험 |
| 프론트엔드 | **S3 + CloudFront** | 서버 관리 불필요, CDN + HTTPS 자동, 콘솔에서 직접 생성 |
| 데이터베이스 | **RDS (MySQL)** | 관리형 DB, 백업 자동 |

### 전체 아키텍처

![전체 아키텍처](/images/aws-architecture.svg)

```
사용자 → CloudFront → S3 (React 정적 파일)
   ↓
사용자 → ALB → EC2 (Spring Boot Docker 컨테이너)
                      ↓
                    RDS (MySQL)
```

> 백엔드 인프라는 **CloudFormation**으로 자동 생성됩니다. 프론트엔드(S3 + CloudFront)는 AWS 콘솔에서 직접 생성합니다.
